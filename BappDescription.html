Click-Jackalope generates ready-to-use HTML iframe proof-of-concept (POC) pages for testing clickjacking vulnerabilities. It enables testers to quickly create and open POCs directly from within Burp Suite without leaving the workflow.

Features

  • Create HTML iframe clickjacking POCs directly from Burp Suite
  • Adds a "Click-Jackalope" tab for manual URL entry and POC generation
  • Right-click context menu option: "Create Click-Jackalope POC" from selected requests
  • Auto-generates filenames based on target host for organized evidence
  • Optionally opens the generated file in your default system browser or Burp's embedded browser
  • Escapes embedded URLs to prevent HTML injection and operates fully offline
  • Lightweight and dependency-free (pure Jython implementation; no external network calls)

Usage

  1. Navigate to the Click-Jackalope tab
  2. Enter or paste the target URL, or right-click a request and choose "Create Click-Jackalope POC"
  3. Optionally edit the output filename
  4. Click Generate to create a local HTML file, or Generate & Open to open it immediately in your browser
  5. Use the generated page to verify whether the target allows framing (absence of X-Frame-Options or permissive Content-Security-Policy: frame-ancestors is demonstrated by a loaded iframe)

Setup

  1. Ensure you have a Jython standalone JAR available (recommended: jython-standalone-2.7.2.jar or later). Download it from the official Jython project and store it locally.
  2. Open Burp → Extender → Options → Python Environment and add the Jython standalone JAR you downloaded. This tells Burp where to find the Python runtime for Jython-based extensions.
  3. Open Burp → Extender → Extensions → Add:
    • Extension type: Python
    • Extension file: select click_jackalope_burp.py from the repository
    After loading you should see a new tab called Click-Jackalope.
  4. If your Burp installation does not expose an unloading handler or certain APIs (older Burp versions), the extension will still function; the extension checks for available APIs at runtime and uses safe fallbacks.
  5. No additional libraries or network access are required; the extension works offline once loaded.

Compatibility & Troubleshooting

  • Burp & Jython: Tested with Burp classic extension API and Jython 2.7.x. If Burp cannot load Python extensions, verify the Python Environment setting and the Jython JAR path.
  • Opening generated files: The extension attempts several methods to open the generated HTML (Java Desktop.browse, Burp callbacks.openUrl, and OS-level open/xdg-open/start). On headless JVMs or restricted environments automatic opening may not succeed — the file is still saved and the path is shown in a dialog.
  • Security: The extension treats target URLs as untrusted and performs basic HTML escaping before embedding them. Always run Click-Jackalope only against targets you are authorized to test.

Notes

  • This extension is intentionally minimal and auditable. If you would like optional features (for example: toggleable sandbox attributes, automatic screenshot capture, or saving POCs into the Burp project workspace), those can be added in follow-up releases.

Author Note

Click-Jackalope was developed to streamline creating evidence artifacts for clickjacking testing during authorized security assests.
